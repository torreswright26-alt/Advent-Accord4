# Canon Policy

## Definitions

### Canon
Explicitly approved narrative, visual, technical, or design information.

### Candidate
Material under consideration but not yet approved.

### Rejected
Material that should not guide future work or enter training.

### Benchmark
A small subset of approved references that most strongly define the target.

### Holdout
Approved references intentionally excluded from training and reserved for evaluation.

## Approval Behavior
- Approval does not authorize redesign.
- Marking an image canon or candidate requires cataloging only unless a new generation is explicitly requested.
- Approved assets remain canon until replaced by explicit instruction.
- Superseded canon is archived rather than deleted.

## Conflict Resolution
Use:
1. Current explicit user instruction
2. Latest approved canon
3. Latest approved benchmark
4. Documentation
5. Prior implementation
